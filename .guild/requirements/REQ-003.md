# Daily Evaluation

## Summary

Implement a "Daily Evaluation" feature called the "Distortion Check" that runs automatically at midnight or when a user clicks "Conclude Day" in the Record of Orders.

The evaluation computes the ratio of Executed vs Diverged tasks for the current day and shows thematic feedback:

- 100% Executed:
  - "Flawless Execution - The Will is Absolute."
  - Show a pristine, glowing seal.
- More than 50% Executed:
  - "Status: Stable Proselyte/Proxy."
  - Show a stable terminal read-out.
- Less than 50% Executed:
  - "Distortion Warning: Deviation levels critical. The Proxies have been dispatched."
  - Trigger a glitchy visual overlay effect.

After evaluation, reset the daily Record of Orders list for the next day while preserving:

- Total Proximate Trust (EXP)
- Custom lists

When the user is distorting, replace the background behind glyphs with `images/distort-carmen.png` at baseline opacity 30%, increasing opacity as rejected prescripts increase:

- 5 failed prescripts = 30%
- 6 failed prescripts = 40%
- and so on

## User Stories

1. As a player, I want a daily Distortion Check to summarize my compliance so each play session ends with clear narrative feedback.
2. As a player, I want the Distortion Check to run automatically at midnight and manually via "Conclude Day" so I can end the day on demand.
3. As a player, I want thematic outcomes based on Execute vs Diverge ratio so consequences reflect performance.
4. As a player, I want the daily Record of Orders cleared after evaluation without losing total trust or custom prescript lists.
5. As a player, when I am distorting, I want the background behind glyphs to switch to `distort-carmen.png` with opacity scaling by rejected count.

## Acceptance Criteria

1. Distortion Check can be triggered by:
   - automatic day boundary crossing (midnight)
   - explicit "Conclude Day" user action in Record of Orders
2. Evaluation uses current-day entries only and computes Execute ratio:
   - 100% Execute: "Flawless Execution - The Will is Absolute."
   - >50% Execute: "Status: Stable Proselyte/Proxy."
   - <50% Execute: "Distortion Warning: Deviation levels critical. The Proxies have been dispatched."
3. Post-evaluation reset clears only current-day Record of Orders entries.
4. Total Proximate Trust and custom prescript lists remain unchanged after reset.
5. Distortion visual mode swaps glyph background to `images/distort-carmen.png`.
6. Distortion opacity baseline and scaling:
   - 5 failed prescripts = 30%
   - each additional failed prescript adds 10% opacity (6=40%, 7=50%, ...)
   - opacity is clamped to a defined max (to be finalized during planning/implementation).
7. Tests cover threshold boundaries (`100%`, `>50%`, `<50%`) and data-reset behavior.

## Edge Cases

- No tasks completed in a day (0/0) must map to a deterministic outcome.
- Midnight crossing while app is closed should still trigger evaluation on next open for prior day data.
- Multiple Conclude Day clicks should not duplicate resets or stack overlays.
- Time zone assumptions must be explicit (local device day boundary).
- Distortion overlay should gracefully handle missing image asset fallback.

## Technical Considerations

- Introduce a day-partition utility to isolate current-day history versus retained global trust state.
- Persist last-evaluated day key to prevent duplicate midnight evaluations.
- Model distortion state derived from daily failed count and/or evaluation result, not ad hoc UI flags.
- Keep evaluation logic pure and unit-testable (input history -> outcome + reset plan).
- Ensure background layer composition does not regress existing cipher/glyph rendering performance.

## Out of Scope

- New rank system mechanics tied to daily outcomes.
- Server-side leaderboards or social sharing of evaluation result.
- Reworking unrelated UI sections outside evaluation and distortion presentation.
