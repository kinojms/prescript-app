# Prescript Difficulty and Scoring Overhaul

## Summary

Update the task generation engine and the Proximate Trust (EXP) state logic to match these exact parameters:

### 1. Updated Difficulty & Trust Scale
- Easy Tier: Rewards 1 Proximate Trust (EXP)
- Medium Tier: Rewards 5 Proximate Trust (EXP)
- Hard Tier: Rewards 10 Proximate Trust (EXP)

### 2. Calibrated Task Pool Injection
Replace placeholder task arrays with a robust default collection of at least 15 default Prescripts distributed across Easy, Medium, and Hard tiers.

The examples below must match exactly:

- Easy:
  - "Stare blankly at the ceiling for exactly 60 seconds."
  - "Perform a full-body standing stretch for 1 minute."
  - "Consume a full glass of clear water immediately."
- Medium:
  - "Drop and execute 5 disciplined push-ups."
  - "Read a news article to observe the status of the City."
  - "Cook a proper meal for your evening sustenance."
- Hard:
  - "Conclude your next bathroom visit without utilizing toilet paper."
  - "Sustain yourself strictly on white rice for your next meal."
  - "Abstain entirely from using AI assistance or language models for the next 4 hours."
  - "Maintain perfect balance on one foot for 5 consecutive minutes."
  - "Recite the alphabet completely in reverse, scattered randomly into your next human conversation without allowing an interruption."

### 3. State Adaptation
Ensure task rendering dynamically pulls the difficulty label and maps precise updated point values to Execute. If source toggles change, custom tasks must also fit this Easy/Medium/Hard framework.

Requested delivery includes complete updated files containing array definitions, selection logic, and scoring state handlers.

## User Stories

1. As a player, I want each prescript difficulty tier (`Easy`, `Medium`, `Hard`) to award fixed Proximate Trust values so progression feels predictable.
2. As a player, I want the default prescript pool to include at least 15 themed directives spread across the three tiers so gameplay variety is sufficient.
3. As a player, I want the exact user-provided directive samples to appear unchanged in the default pool so tone and narrative intent are preserved.
4. As a player, I want the active task card to display difficulty and award the exact mapped trust payout when I click Execute.
5. As a player, I want custom prescripts to remain compatible with source toggles while still conforming to `Easy/Medium/Hard` difficulty options.
6. As a maintainer, I want scoring constants and selection logic centralized so future balancing changes are low-risk.

## Acceptance Criteria

1. Difficulty payout mapping is exactly:
   - Easy = 1 trust
   - Medium = 5 trust
   - Hard = 10 trust
2. Default prescript dataset contains at least 15 entries with every entry explicitly labeled `Easy`, `Medium`, or `Hard`.
3. The listed example directives from user input are present verbatim in the default pool.
4. Execute action calculates trust using the selected prescript's difficulty-to-payout map, not hardcoded per-screen values.
5. Rendering layer reads and displays difficulty label dynamically from prescript data.
6. Custom prescript creation/edit flows restrict difficulty selection to `Easy`, `Medium`, or `Hard`.
7. Source toggles (`default/custom`) continue to function with no regressions in prescript selection.
8. Tests cover payout mapping and at least one happy-path execute action per tier.

## Edge Cases

- Invalid or missing difficulty on legacy localStorage entries should safely normalize to `Easy` (or be filtered) to prevent crashes.
- If source toggles produce an empty prescript pool, app should fail gracefully (existing no-directive fallback remains intact).
- Backup import with outdated difficulty values should be normalized or rejected with a clear message.
- Existing history records should remain readable even after payout table changes.

## Technical Considerations

- Keep trust payout map in a shared constant (single source of truth) used by reducer/action handlers.
- Ensure difficulty typing aligns across default data, custom data, and hydration validation.
- Update tests in state hook and prescript selection modules to include `Medium` tier paths.
- Confirm any UI strings and filters that previously assumed two-tier difficulty are upgraded to three tiers.

## Out of Scope

- Changing rank thresholds or rank titles.
- Reworking visual design unrelated to difficulty display.
- Migrating historical trust already earned by prior versions.
