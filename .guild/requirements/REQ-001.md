# GitHub Pages Deployment

## Summary

Deploy the prescript-app (React + Vite PWA) to GitHub Pages at https://kinojms.github.io/prescript-app/ with automated CI/CD via GitHub Actions on every push to `master`.

## Spec

docs/superpowers/specs/2026-06-14-github-pages-deploy-design.md

## User Stories

- As a user, I can access the app at https://kinojms.github.io/prescript-app/
- As a developer, pushing to `master` automatically deploys the latest build

## Acceptance Criteria

- [ ] App loads correctly at https://kinojms.github.io/prescript-app/
- [ ] All assets (JS, CSS, audio, icons) load without 404s
- [ ] PWA install prompt works correctly (correct scope and start_url)
- [ ] GitHub Actions workflow runs green on push to master

## Technical Considerations

- Vite `base` must be set to `/prescript-app/` for sub-path asset resolution
- `public/manifest.json` paths must be manually updated (Vite does not rewrite public/ files)
- GitHub repo Settings → Pages → Source must be set to "GitHub Actions" (one-time manual step)

## Out of Scope

- Custom domain configuration
- Preview deployments for PRs
- Environment-specific builds
